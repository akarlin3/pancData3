# pancData3 — Graph FAQ

A reference document covering every figure that the pancData3 pipeline + analysis report can produce, organized in the order they appear in the PDF report. For each figure, four questions are answered:

- **What it shows** — literal description.
- **How it's generated** — code path, function, key parameters.
- **What it means** — clinical and statistical interpretation.
- **Caveats** — known limitations, sharp edges, things not to over-interpret.

**Audience:** the successor researcher (who will rerun the pipeline) and the PI / committee reviewer (who will read the report without rerunning). Code paths are kept terse; interpretation is the priority.

**Conventions used throughout:**

- *DWI strategy* = one of `Standard`, `dnCNN`, `IVIMnet`. Most figures are produced once per strategy.
- *LC* = local control (no in-field recurrence). *LF* = local failure. *Competing risk* = non-cancer death without prior LF (encoded `lf==2`).
- *Fx1* = baseline (fraction 1). *Fx2–Fx5* = subsequent fractions of the 5-fraction MR-Linac course.
- *Core method* = one of the 11 tumor-core delineation algorithms; `adc_threshold` is the default and the universal fallback.

## Section A — Overview

### A1. Cohort overview table
- **Shows:** Patients × timepoints by DWI strategy. A small-sample warning (red box) appears if max patient count < 30.
- **Generated by:** `analysis/report/sections/enrollment.py::_section_cohort_overview`. Pulled from MAT-file `longitudinal.num_patients` and `longitudinal.num_timepoints`.
- **Means:** Sanity check on cohort size before any inference. Anything < 30 patients limits the inferential strength of the rest of the report.
- **Caveats:** Counts can differ across DWI strategies if a strategy failed to produce fits for some patients (a separate warning fires when |max − min| > 2). If you see counts that disagree, treat cross-DWI comparisons downstream with extra skepticism.

### A2. Outcome balance (LF / LC / competing risk) cards
- **Shows:** Stat cards with LF, LC, and competing-risk patient counts. An imbalance warning fires when the LC:LF ratio exceeds 3:1.
- **Generated by:** `enrollment.py::_section_cohort_overview` (outcome-balance subsection).
- **Means:** Class balance check for the predictive modeling section. Severe imbalance (say 35 LC : 7 LF) means AUC, sensitivity, and specificity must be read with caution and that class-weighted analysis should be considered.
- **Caveats:** Competing-risk patients are reported but excluded from group comparisons and ROC analysis. They re-enter only in the survival models via the cause-specific hazards framework.

### A3. Patient flow (CONSORT-style) table
- **Shows:** Attrition at each pipeline stage: initial cohort → baseline exclusion → outlier removal → competing-risk exclusion.
- **Generated by:** `enrollment.py::_section_patient_flow`. Combines MAT-file counts with parsed log fields (`baseline_exclusion`, `total_outliers`, `glme_excluded`).
- **Means:** Tells you how the analysis cohort was reached from the screening cohort. Standard publication requirement.
- **Caveats:** A "parse warnings" box appears if the parser couldn't fully read the logs — when present, the numbers may be incomplete. Always cross-check against the MATLAB diary files in `saved_files_*/` if anything looks off.

## Section B — Data quality and completeness

### B1. Data quality summary
- **Shows:** Per-DWI counts of NaN voxels, Inf voxels, negative-ADC voxels, and outlier patients flagged by the sanity checker.
- **Generated by:** `pipeline/core/sanity_checks.m` produces the diary; `enrollment.py::_section_data_completeness` parses it.
- **Means:** Flags upstream problems (bad fits, registration failures, motion artifacts). High NaN fractions on a single patient usually indicate a fit failure; high counts on the same fraction across patients usually indicate a protocol issue.
- **Caveats:** The 50% in-mask NaN-fraction warning threshold is fixed in `sanity_checks.m`. A patient at 49% NaN fraction is not flagged but is borderline.

### B2. Completeness matrix
- **Shows:** Patient × timepoint matrix marking which fractions have valid data per DWI strategy.
- **Generated by:** `enrollment.py::_section_data_completeness`.
- **Means:** Where the missing-data is concentrated. Patterns matter: random missingness is fine; a column missing for 20% of patients suggests a fraction-specific protocol or scanner issue.
- **Caveats:** "Valid" here means the fit converged, not that the result is clinically interpretable. Combine with the data quality summary (B1) to judge.

## Section C — Parameter maps (per DWI strategy)

### C1. Parameter Maps overlaid on Anatomy (paginated multi-patient figures)
- **Shows:** Three panels per patient — (a) b=0 anatomical image with GTV contour, (b) full-slice ADC map with GTV contour, (c) ADC overlaid on anatomy inside GTV only. Patients are batched 4–5 per figure.
- **Generated by:** `pipeline/core/visualize_results.m::plot_parameter_maps_streaming` (and the standalone `plot_parameter_maps.m`). Output: `parameter_maps_NN.png` at 300 DPI.
- **Means:** Spatial QA. Three things to look for: contour-vs-anatomy alignment (misregistration corrupts everything downstream); spatial heterogeneity (focal low-ADC regions = potential treatment-resistant subclones); geometric distortion at tissue–air boundaries near the pancreas (a known artifact at 1.5T).
- **Caveats:** Only Fx1 anatomy is shown by default. The pipeline skips parameter map generation if the b-value protocol deviates from the expected `[0 30 150 550]` set — protocol deviations invalidate the monoexponential ADC fit because b-values < 200 capture perfusion contributions that a single-component ADC model cannot disentangle.

## Section D — Feature distributions (per DWI strategy)

### D1. Feature histograms (LC vs LF, 4-panel: ADC, D, f, D*)
- **Shows:** Overlaid LC (blue) vs LF (red/orange) histograms for each of the four baseline biomarkers at Fx1.
- **Generated by:** `visualize_results.m::plot_feature_distributions_streaming` → `plot_feature_distribution.m`. Output: `Feature_Histograms_<strategy>.png`.
- **Means:** Visual test of whether pre-treatment biomarker distributions differ between outcome groups — the foundational hypothesis of predictive biomarker work. Clear separation suggests a candidate predictor; overlapping distributions argue against it.
- **Caveats:** Histogram bin widths are fixed; small cohorts can produce spurious-looking bimodality. Read alongside the boxplots (D2), which compress bin-choice noise. In Octave, hist() bin semantics differ from MATLAB's — this was fixed in v2.0 but worth knowing if running on Octave.

### D2. Feature boxplots (LC vs LF, 4-panel) with Wilcoxon p-values
- **Shows:** Side-by-side boxes per outcome group for each biomarker, with the Wilcoxon rank-sum p-value annotated.
- **Generated by:** `plot_feature_distribution.m` (boxplot mode). Output: `distributions_boxplots_<strategy>.png`.
- **Means:** Compact statistical comparison. Wilcoxon (non-parametric) is used because IVIM parameters — especially f and D* — are typically non-normal with heavy right tails.
- **Caveats:** Wilcoxon p-values here are *not* multiplicity-corrected (the correction is applied in the Statistics section, not here). A p-value of 0.04 in this panel is not the same as 0.04 after correction. A large boxplot warning fires when only one outcome group has data.

## Section E — Dose–diffusion correlation (per DWI strategy)

### E1. Dose vs Diffusion scatter (3 metrics × 2 dose endpoints = 6 panels)
- **Shows:** Scatter of each of {ADC, D, f} against each of {Mean GTV Dose, D95}. LC patients in blue, LF in red, with per-group linear trend lines and Spearman correlation coefficients annotated. D* is excluded (too noisy).
- **Generated by:** `pipeline/core/plot_scatter_correlations.m` via `visualize_results.m::plot_scatter_correlations_streaming`.
- **Means:** Tests dose–response relationships. Lower ADC/D = denser tumor cellularity = potentially needs higher dose. Lower f = more hypoxic = more radioresistant. A monotonic but nonlinear trend is the expected shape, which is why Spearman is preferred over Pearson here.
- **Caveats:** Per-group trend lines are deliberate — pooled trends suffer from Simpson's paradox if LF and LC have different dose distributions. Competing-risk patients (lf==2) are excluded; a small-group warning fires when the LF group has n<3.

## Section F — Cross-DWI subvolume comparison

### F1. Cross-DWI ADC subvolume agreement (Dice / Hausdorff at Fx1)
- **Shows:** Dice and HD95 agreement between subvolumes defined by the same core method across the three DWI strategies (Standard vs dnCNN vs IVIMnet).
- **Generated by:** `visualize_results.m::plot_cross_dwi_subvolume_comparison_streaming`.
- **Means:** Tests whether the *spatial location* of the resistant subvolume is robust to the choice of denoising/fitting strategy. High Dice across strategies = the resistant region is a real anatomical thing, not an artifact of fitting choice.
- **Caveats:** Only runs when at least two DWI strategies have results on disk; fails gracefully (warning, no figure) otherwise. Hausdorff is only computed when 3D GTV masks are available.

## Section G — Trajectory plots (opt-in, requires `run_trajectory_plots: true`)

### G1. Waterfall plot — best ADC response per patient
- **Shows:** Bar chart of patients sorted by best ADC % change from baseline. Green bars ≤ −20% (responders), red bars ≥ +20% (progressors), grey in between.
- **Generated by:** `pipeline/core/visualize_results.m::plot_waterfall_chart`. Output: `waterfall_adc_response.png` and `.eps`.
- **Means:** Standard oncology visualization for response distribution. Tells you what fraction of the cohort moved into the responder/progressor zones.
- **Caveats:** ±20% thresholds are clinical conventions, not data-driven. Patients with NaN at any timepoint are dropped silently. Data is sourced from `calculated_results.percent_deltas.adc_mean`; missing field raises `visualize_results:missingField`.

### G2. Swimmer plot — patient timelines
- **Shows:** Horizontal bars per patient indicating follow-up duration, with markers at each scan day.
- **Generated by:** `visualize_results.m::plot_swimmer_chart`. Output: `swimmer_timeline.png` and `.eps`.
- **Means:** Visual cohort overview of follow-up depth. Useful for spotting patients with very short follow-up that may be censored too early to contribute to survival analysis.
- **Caveats:** Patients are sorted by follow-up length (descending). The bar color is fixed light blue — outcome status is NOT encoded in the bar (separate panel would be needed). Source: `calculated_results.scan_timeline`.

### G3. Spider plot — longitudinal ADC trajectories
- **Shows:** ADC % change from cohort median over time, one line per patient.
- **Generated by:** `visualize_results.m::plot_spider_chart`. Output: `spider_adc_trajectories.png` and `.eps`.
- **Means:** Visualizes individual treatment trajectories. Convergent lines = consistent response; divergent lines = heterogeneous response.
- **Caveats:** Reference is the *cohort* median at baseline, not the patient's own baseline — this is intentional for visualizing relative trajectories but means individual % changes will not match what a patient-centered trajectory would show. Zero-baseline guard is implemented (denominator floored at 1).

## Section H — Statistics by graph type

### H1. Trend agreement across DWI strategies
- **Shows:** Percentage of biomarker trends (e.g., "ADC increases over time") that agree across Standard / dnCNN / IVIMnet.
- **Generated by:** `analysis/report/sections/manuscript_results.py::_compute_all_groups_trend_agreement` and rendered in the cross-DWI section.
- **Means:** ≥80% agreement = "high robustness"; 60–80% = moderate; <60% = limited. Direct measure of whether the choice of DWI strategy materially changes the story.
- **Caveats:** Trend direction is computed as a discrete classification (increasing / decreasing / stable). A series that is "increasing" by a hair in one strategy and "stable" by a hair in another counts as disagreement, even though the underlying numbers may be nearly identical.

### H2. Significance summary table
- **Shows:** All p-values from group comparisons across DWI strategies, colored by significance level.
- **Generated by:** `analysis/parsers/test_statistical_relevance.py` extracts; the report renders.
- **Means:** A consolidated view of which biomarker × strategy × timepoint combinations are statistically distinguishable between LC and LF.
- **Caveats:** Both raw and Bonferroni-corrected columns are shown — read the corrected one for any inferential claim. The number of tests is large (4 biomarkers × 3 strategies × ~5 timepoints), so Bonferroni is conservative but appropriate.

### H3. Effect sizes
- **Shows:** Cohen's d (or equivalent) for each comparison alongside the p-value.
- **Generated by:** Same parser stack as H2.
- **Means:** A way to read significance independent of sample size. Small p with small d = real but tiny effect; large d with large p = potentially meaningful but underpowered.
- **Caveats:** With n ≈ 42 split between LC/LF, effect sizes here have wide confidence intervals. Interpret directionally, not as point estimates.

## Section I — Multiple-comparison corrections

### I1. FDR-corrected significance (global)
- **Shows:** Benjamini–Hochberg FDR-corrected p-values across all hypothesis tests in the report.
- **Generated by:** `report/sections/` FDR section.
- **Means:** Controls the false discovery rate at 0.05 across the full test family. The right correction to cite when claiming "significant after correction" in a manuscript with many tests.
- **Caveats:** FDR is computed across all tests treated as one family, which is conservative for this design. A subset analysis (e.g., FDR within just the dosimetry tests) would be more lenient and is sometimes a defensible alternative — but is not what's reported by default.

## Section J — Correlations

### J1. Correlation matrix / heatmap
- **Shows:** Pairwise correlations among biomarkers and dose endpoints.
- **Generated by:** Report `correlations` section.
- **Means:** Used for collinearity diagnostics ahead of the predictive modeling. High correlation between two features means the elastic net will not be able to assign both stable coefficients.
- **Caveats:** Correlation does not imply causation — there's a standing causation caveat info-box in the rendered report. Do not over-read off-diagonal values without checking the underlying scatter (E1).

## Section K — Treatment / dose coverage

### K1. D95 ADC pass/fail check (vs 45 Gy threshold) — per DWI strategy
- **Shows:** Bar or row marker indicating whether each strategy's D95 ADC sub-volume coverage meets the 45 Gy threshold.
- **Generated by:** `analysis/report/sections/data_supplemental.py` — pass/fail check from MAT-file `dosimetry.d95_adc_mean`.
- **Means:** **This is the headline finding for the AAPM abstract.** D95 ≈ 36.9 Gy means the ADC-defined resistant region receives roughly 26% less dose than the radical-intent threshold. Motivates daily adaptive replanning targeted at the diffusion-defined sub-volume.
- **Caveats:** 45 Gy is a clinical convention, not derived from this cohort. The pass/fail status would change if a different threshold were used. The result is robust across DWI strategies (this is part of why it is the lead finding) but the threshold choice itself is an external assumption.

### K2. Dosimetric interpretation block (V50, D95 across DWI strategies and parameters)
- **Shows:** Per-strategy and per-parameter (ADC, D, f, D*) D95 and V50 values with concern callouts when below thresholds.
- **Generated by:** `analysis/report/sections/main_results_hypothesis.py` builds dosimetry section; `data_supplemental.py` renders.
- **Means:** Confirms that the dose shortfall is observed for ADC-defined and D-defined sub-volumes consistently — i.e., it's a property of the underlying anatomy, not an ADC artifact.
- **Caveats:** V50 is reported as a fraction in some upstream paths and as a percentage in others; the renderer normalizes via `_scalar_gy` and a < 1.0 heuristic. If you ever see V50 = 0.69 instead of 69%, that's the heuristic catching a not-yet-normalized value, not a real bug.

## Section L — Predictive modeling

### L1. ROC curve with Youden cutoff (per timepoint, per DWI strategy)
- **Shows:** ROC curve from LOOCV out-of-fold elastic-net risk scores. Annotated with AUC, the Youden-optimal operating point, sensitivity, and specificity at that point.
- **Generated by:** `pipeline/utils/plot_predictive_diagnostics.m`. Output: `ROC_OOF_Risk_Score_<fx>_<strategy>.png`.
- **Means:** Discriminative ability of the multivariate predictive model on truly held-out patients. AUC ≥ 0.80 = high; 0.70–0.80 = good; 0.65–0.70 = moderate; below that = not clinically useful.
- **Caveats:** LOOCV is the right design for this small cohort (n ≈ 42) but variance on the AUC estimate is still high — a bootstrap 95% CI is reported alongside. With sub-50 patients and 22 candidate features, the Youden cutoff in particular is not stable across resampling and should not be quoted as a clinical decision threshold without external validation. Competing-risk patients are excluded from this curve (otherwise they'd inflate the false-positive rate).

### L2. Sanity check panels (per selected feature)
- **Shows:** Three panels per selected feature — (1) GTV volume vs feature value (volume confounder check), (2) ADC SD heterogeneity, (3) signal-vs-noise scatter with the test–retest CoR band overlaid.
- **Generated by:** `plot_predictive_diagnostics.m`. Output: `Sanity_Checks_*.png`.
- **Means:** Verifies the selected biomarker is not confounded. Volume panel = is the signal an artifact of partial-volume contamination from smaller tumors having more edge voxels? SD panel = is heterogeneity itself the signal? CoR band = is the apparent change larger than scanner noise?
- **Caveats:** The CoR band uses test–retest data from `compute_repeatability` if available, otherwise falls back to a literature-derived value (~0.4 × 10⁻³ mm²/s for abdominal DWI). The fallback is approximate; values within the band should not be over-interpreted.

### L3. 2D feature space scatter (with logistic decision boundary)
- **Shows:** Pairwise scatter of selected features colored by outcome, with the logistic decision boundary overlaid.
- **Generated by:** `plot_predictive_diagnostics.m`. Output: `2D_Space_*.png`.
- **Means:** Geometric intuition for why the multivariate model separates LC from LF when no single feature does. Diagonal boundaries indicate genuine multivariate gain; near-axis boundaries mean one feature is doing all the work.
- **Caveats:** With n ≈ 42, decision-boundary placement is sensitive to a few patients near the margin. Visual separation in this panel is not the same as out-of-sample discrimination — that's what L1 is for.

### L4. Forest plot of hazard ratios (Cox PH)
- **Shows:** HR with 95% CI for each covariate in the Cox model, log-scale x-axis, with a vertical reference at HR = 1.
- **Generated by:** `analysis/report/sections/manuscript_findings.py` extracts HRs from the Cox section; report renderer builds forest plot.
- **Means:** Which features increase or decrease local-failure hazard, and by how much. CI crossing 1 = not statistically significant.
- **Caveats:** HRs from Cox PH assume the proportional hazards assumption holds. The PH check (Schoenfeld residuals) is in M3. When PH is violated, time-varying Cox results (M2) supersede these.

## Section M — Model diagnostics and robustness

### M1. Calibration plot
- **Shows:** Predicted risk vs observed risk by decile. Points should fall near the y = x diagonal.
- **Generated by:** `compute_calibration_metrics.m` (Brier score, Hosmer–Lemeshow test, calibration slope/intercept). Rendered in the model diagnostics section.
- **Means:** A well-discriminating model can still be miscalibrated. Calibration matters when you intend to use the predicted probability as a risk estimate (e.g., to set a treatment threshold) rather than just to rank patients.
- **Caveats:** With n ≈ 42, deciles aren't really deciles. The H-L test is underpowered at this size; do not over-rely on its p-value.

### M2. Time-varying Cox results
- **Shows:** Stratified Cox model output and extended-Cox interaction terms (covariate × log(time)) for any covariate that violated PH in M3.
- **Generated by:** `pipeline/utils/fit_time_varying_cox.m`. Parsed by `analysis/parsers/test_parse_imputation_and_tv_cox.py`.
- **Means:** When PH is violated, the standard Cox HR is not a single number — the hazard ratio changes over time. The interaction term tells you the direction of that change.
- **Caveats:** Only triggered when `fit_time_varying_cox: true` in config. The time-stratification choice (median split on the violating covariate) is a heuristic; a clinically-motivated stratum boundary would be better when one is available.

### M3. Schoenfeld residual plots (PH assumption diagnostic)
- **Shows:** Scaled Schoenfeld residuals vs time for each Cox covariate, with a LOWESS smoother. Flat = PH holds; trending = PH violated.
- **Generated by:** `pipeline/utils/compute_schoenfeld_residuals.m`. Output: residual diagnostic figures.
- **Means:** Tests whether each covariate's effect is truly constant over follow-up. Trending residuals indicate the M2 time-varying analysis is required.
- **Caveats:** LOWESS span is fixed; visual interpretation is partly subjective. A formal Spearman correlation p-value is reported alongside.

### M4. Imputation sensitivity comparison
- **Shows:** AUC under each of four imputation strategies — KNN (default), LOCF, mean, linear interpolation.
- **Generated by:** `pipeline/utils/imputation_sensitivity.m`. Parsed by `test_parse_imputation_and_tv_cox.py`.
- **Means:** Robustness of the predictive AUC to the choice of imputation method. Tight clustering across methods = the result is not driven by an imputation artifact.
- **Caveats:** Only triggered when `run_imputation_sensitivity: true`. KNN is the default in the pipeline. If LOCF AUC differs substantially from KNN AUC, the predictive signal may be partially driven by the temporal fill choice and warrants closer inspection.

### M5. Decision curve analysis (DCA)
- **Shows:** Net benefit of the model across threshold probabilities, vs "treat all" (red dashed) and "treat none" (black solid) reference strategies.
- **Generated by:** `pipeline/utils/decision_curve_analysis.m`. Output: `dca_<strategy>_<fx>.png`.
- **Means:** Is the model clinically useful at any plausible decision threshold? The model is useful only at thresholds where its curve sits above both references. The "useful range" annotation gives the threshold interval where this holds.
- **Caveats:** DCA assumes the threshold probability reflects clinically meaningful tradeoff between false-positive and false-negative costs. With small cohorts the curves are noisy and the useful range can shift on resampling.

### M6. Net reclassification improvement (NRI / IDI) plot
- **Shows:** Continuous and category-based NRI between the predictive model and a baseline (typically GTV-volume-only).
- **Generated by:** `pipeline/utils/compute_nri.m`. Reported in model robustness section.
- **Means:** How many patients are correctly reclassified up (LF) or down (LC) when the model adds DWI features over the baseline. Positive NRI = the DWI features add information.
- **Caveats:** NRI can be inflated by overfitting if the model is not externally validated. Read alongside the L1 LOOCV AUC.

## Section N — Survival curves (cause-specific hazards)

### N1. Kaplan–Meier curves stratified by predicted risk
- **Shows:** Two KM curves — high-risk vs low-risk (split at median LOOCV risk score) — with log-rank p-value and at-risk table.
- **Generated by:** `pipeline/core/metrics_survival.m`. Output: KM PNG saved to the strategy-specific folder.
- **Means:** Translates the predictive AUC into a clinically interpretable survival difference. Wide separation = the model identifies a high-risk group with materially worse local control.
- **Caveats:** Median-split risk dichotomization throws away information; the continuous Cox HR (L4) is more statistically efficient. Competing-risk patients are handled via cause-specific hazards — see N2 for the alternative framing.

### N2. Cumulative incidence curves (Fine–Gray subdistribution hazards)
- **Shows:** Cumulative incidence of local failure over time, with competing-risk death properly accounted for via subdistribution weights.
- **Generated by:** `metrics_survival.m` Fine–Gray section.
- **Means:** The right way to display "probability of LF by time t" when competing risks are present. Standard 1−KM would overestimate this probability because it treats competing-risk deaths as censored rather than as preventing future LF.
- **Caveats:** The Fine–Gray model uses proper subdistribution weights (the ad-hoc 0.5 multiplier was fixed in v2.1.0; if you see old code or old reports referencing it, those are stale). IPCW-weighted standard errors use a sandwich variance ratio (also v2.1.0 fix).

### N3. Schoenfeld residuals — Fine–Gray
- **Shows:** Same as M3 but for the Fine–Gray model rather than cause-specific Cox.
- **Generated by:** `compute_schoenfeld_residuals.m`.
- **Means:** PH check for the subdistribution hazards model.
- **Caveats:** PH for Fine–Gray means the *subdistribution* hazard is proportional, which is a slightly different assumption from cause-specific PH. A covariate can satisfy one and violate the other.

## Section O — Cross-method (core method) comparison — opt-in (`compare_cores`)

### O1. Pairwise Dice heatmap (11 × 11)
- **Shows:** Mean Dice coefficient between each pair of the 11 tumor-core methods, averaged across patients and timepoints.
- **Generated by:** `pipeline/core/compare_core_methods.m`. Output: `core_method_dice_heatmap_<strategy>.png`.
- **Means:** How spatially similar are the masks produced by different core-definition methods? Mean off-diagonal Dice ≥ 0.7 = good agreement; 0.5–0.7 = moderate; below 0.5 = method choice strongly impacts which voxels are labeled core, which means downstream sub-volume metrics may differ materially across methods.
- **Caveats:** Mean Dice averages over patients with very different tumor sizes; small tumors push Dice down mechanically. The colormap was switched to `parula` in v2.0 to fix yellow-on-yellow contrast issues.

### O2. Pairwise Hausdorff distance (HD95) heatmap
- **Shows:** Same matrix as O1 but with 95th-percentile Hausdorff distance instead of Dice.
- **Generated by:** `compare_core_methods.m`. Output: `core_method_hd95_heatmap_<strategy>.png`. Only generated when 3D GTV masks are available.
- **Means:** Complements Dice. Two methods can have moderate Dice but small HD95 (consistent boundary, slightly different volumes) or high Dice and large HD95 (similar overall but with one method producing isolated outlier voxels).
- **Caveats:** HD95 is in voxel or millimeter units depending on the underlying mask. Check the colorbar label.

### O3. Volume comparison bar chart (per method)
- **Shows:** Mean core volume as a percentage of GTV, with error bars, one bar per core method.
- **Generated by:** `compare_core_methods.m`. Output: `core_method_volume_comparison_<strategy>.png`.
- **Means:** Sanity check that the methods are producing similar-sized cores. Wildly different volumes = the methods are answering different questions and shouldn't be aggregated.
- **Caveats:** A method with a low mean volume but a high failure-rate might be silently falling back to `adc_threshold` for many patients, dragging its volume distribution toward the threshold method's. Check O4 alongside this.

### O4. Fallback rate bar chart (per method)
- **Shows:** Per-method fraction of patient × timepoint instances where the method failed and fell back to `adc_threshold`.
- **Generated by:** `compare_core_methods.m`. Output: `core_method_fallbacks_<strategy>.png`.
- **Means:** Reliability check. Methods with high fallback rates contributed less unique information to the analysis even when included. Drives the WS2 pruning decision.
- **Caveats:** **Important context for the v2.4 pipeline:** failure-rate computation uses `min_core_voxels = 10`, NOT `min_vox_hist = 100`. The `min_vox_hist` value was historically used and inflated apparent failure rates. The pruning cutoff `max_core_failure_rate = 0.25` (locked) operates on the corrected definition; 6 of 11 methods survive. `adc_threshold` itself is always retained regardless of cutoff.

## Section P — Texture features (opt-in)

### P1. Texture feature heatmap or table
- **Shows:** GLCM-derived texture features (contrast, homogeneity, energy, entropy, correlation) per patient, stratified by outcome.
- **Generated by:** `pipeline/utils/compute_texture_features.m`. Reported in the texture section.
- **Means:** Higher-order tumor heterogeneity descriptors. Texture features can capture spatial patterns that mean ADC misses.
- **Caveats:** Texture features are highly sensitive to voxel size, masking, and quantization choices. They were added in v2.1.0 and are best treated as exploratory until externally replicated. Errors are logged with message IDs but the section may be empty if GLCM computation failed.

## Section Q — Registration quality

### Q1. Registration QA summary
- **Shows:** Per-patient registration quality metrics: NMI, motion-artifact flags, alignment residuals.
- **Generated by:** `pipeline/utils/detect_motion_artifacts.m` flags; reported in the registration QA section.
- **Means:** Registration failures invalidate longitudinal analysis at the affected timepoint. This panel is where you spot patients who should probably be excluded from the longitudinal models even though they passed the basic sanity checks.
- **Caveats:** NMI is flagged when it returns NaN (insufficient voxel overlap). The fixed thresholds for motion flagging are conservative — borderline patients should be reviewed manually before exclusion.

## Section R — Sensitivity and power

### R1. Power analysis chart
- **Shows:** Detectable effect size as a function of sample size for the primary endpoints, with the actual cohort size marked.
- **Generated by:** Report `power` section.
- **Means:** Honest communication of statistical resolution. Anchors the discussion of null findings.
- **Caveats:** Power calculations assume effects of a particular type (two-sample test, fixed alpha). They are illustrative, not definitive.

### R2. Sensitivity analyses
- **Shows:** Headline metrics (AUC, HR) recomputed under perturbed analysis choices: outlier inclusion, alternative imputation, alternative core method.
- **Generated by:** Report `sensitivity` section, drawing on M4 and parallel runs.
- **Means:** Robustness check. A finding that holds across all sensitivity perturbations is more credible than one that depends on a specific analytical choice.
- **Caveats:** Sensitivity here means analytical sensitivity, not biological sensitivity. Does not protect against biases in the underlying data or cohort selection.

## Section S — Appendix

### S1. Figure gallery
- **Shows:** Thumbnail grid of every figure generated by the pipeline run, with captions.
- **Generated by:** `analysis/report/sections/gallery.py`.
- **Means:** A visual index to navigate to specific figures.
- **Caveats:** The gallery includes some failed/empty figures with placeholder captions. Use it as an index, not as a curated set.

### S2. Figure issues panel
- **Shows:** Auto-detected rendering problems (missing axis labels, truncated data, low-resolution warnings, low-sample annotations).
- **Generated by:** `analysis/parsers/batch_graph_analysis.py` vision-model analysis.
- **Means:** A reviewer-style checklist of figure quality issues. Worth scanning before the report goes to a PI or external collaborator.
- **Caveats:** Vision-model output is not perfect. Treat the issues list as a starting point for review, not as ground truth.

### S3. Reporting checklist (TRIPOD / RECORD-style)
- **Shows:** A structured checklist of which reporting items have been addressed in this report run.
- **Generated by:** `report/sections/publication.py`.
- **Means:** Submission-readiness check. Each unchecked item is a manuscript revision target.
- **Caveats:** The checklist is descriptive of what's *in the report*, not normative — being checked does not mean the item is well-addressed, only that the corresponding report section is non-empty.

### S4. Table index and figure index
- **Shows:** Lists of every table and every figure in the report with anchor links.
- **Generated by:** `report/sections/publication.py` and `report_formatters.py`.
- **Means:** Navigation aid for long reports. Useful when the PDF is 80+ pages.
- **Caveats:** Anchor links work in HTML; some PDF viewers don't honor them. The figure index is alphabetical, not in report order.

## Section T — Things this FAQ does *not* cover

- Per-patient anatomical figures (covered en masse in C1, but individual outliers are not curated).
- Diagnostic figures emitted only on failure (e.g., parameter map skipped due to protocol deviation — the lack of a figure is itself the diagnostic).
- Internal scatter / debug figures saved to `pipeline/tests/` during test runs.
- Improvement-loop process visualizations (those live in iteration logs, not the patient-facing report).
